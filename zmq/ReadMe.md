
测试出现的问题或现象：
1、zeromq在插拔网线中容易出现半开连接的现象，出现这种现象需要配合上层心跳做重连和KEEP_ALIVE可以解决。
2、zeromq的重连可以重新连接上。
3、zeromq的服务端没有手动管理的功能，当应用需要管理seesion时，则不适用。容易被没有使用的连接占据连接数。
4、zeromq进程间通信很好用。
5、zeromq可以发送原始流与其它的socket通信，使用STREAM模式，但要注意发送和接收前的ZMQ_IDENTITY.使用STREAM模式的时候，设置的与真正要发送前一帧不同。
6、zeromq作为客户端适用。服务端做为消息队列也适用。但不适用于需要管理的场景。
7、socket close需要时间处理。否则不会产生FIN关闭信号。
8、DEALER模式会自动处理粘包，STREAM模式不会自动处理粘包。

已测试范围：
1、客户端网络断开（禁用或拔网线）
2、服务端网络断开（禁用或拔网线）
3、小片和大片发送（1字节和1000000字节）
4、连续重连（1天测试）
5、自动重连
6、连接和接收数据回调
7、随机连接发送（>10000）
8、速率测试（DEALER百兆网络750KB/S）（STREAM百兆网1.5MB/S），跟发送的数据大小有关
9、多线程安全
10、重置IP和端口
